## device 해상도 기준
Device의 해상도를 선택하는 기준은 아래의 사이트를 참고 하였다. <br>
<a href="https://gs.statcounter.com/screen-resolution-stats">https://gs.statcounter.com/</a>

### PC (1920px)
statcounter를 확인한결과 작년부터 지금까지 꾸준하게 1920px기준의 web을 많이 사용한것으로 예상되어져 pc의 해상도 기준은 
1920px * 1080px 의 해상도를 기준으로 잡았다.

### Tablet (1024px)
statcounter를 확인한결과 tablet의 경우 다채롭게 사용이 되고있으며, 현제 태블릿 디바이스 중 최대 해상도인 1024*1366px기준으로 잡았다.

### mobile (420px)
statcounter를 확인한결과 mobile은 다채롭게 사용이 되고있어 mobile의 해상도 기준은 가장 큰 사이즈 기준인 
420*640기준으로 잡았다. 


## Device별 그리드 기준
반응형 웹 퍼블리싱의 기본은 '계산'이다. 어떤 width, heihgt의 device를 쓰는 유저가 접속을 해도 컨텐츠 내용이 깨지면 안된다.
따라서 다음과 같은 기준의 그리드를 잡고 퍼블리싱을 진행하고있다.

### PC (12col/1080px)
컨텐츠내용의 집중도를 올리기 위해서 전체의 80%기준으로 그리드를 잡았고, 해당 그리드가 1080px이상 넘어갈 경우 사이트 컨텐츠에 대한 집중도가 많이 떨어져 그리드 최대 값을 1080px 사이즈로 잡았다. <br>
해당 그리드를 12column grid시스템으로 적용하여 반응형 스타일을 매끄럽게 구현할 수 있도록 하였다.<br>
<ul>
  <li>전체 그리드 사이즈 : 80%</li>
  <li>최대 그리드 사이즈 : 1080px</li>
  <li>그리드 컬럼 : 12</li>
</ul>

### Tablet (8col/700px)
컨텐츠내용의 집중도를 올리기 위해서 전체의 90%기준으로 그리드를 잡았고, 해당 그리드가 700px이상 넘어갈 경우 사이트 컨텐츠에 대한 집중도가 많이 떨어져 그리드 최대 값을 700px 사이즈로 잡았다. <br>
해당 그리드를 8column grid시스템으로 적용하여 여러해상도의 디바이스에서도 매끄럽게 구현할 수 있도록 하였다.<br>
<ul>
  <li>전체 그리드 사이즈 : 90%</li>
  <li>최대 그리드 사이즈 : 700px</li>
  <li>그리드 컬럼 : 8</li>
</ul>

### mobile (4col/90%)

해당 그리드를 4column grid시스템으로 적용하여 여러 해상도의 디바이스에서도 매끄럽게 구현할 수 있도록 하였다.<br>
