이번 포스팅에서는 간단하게 시멘틱 태그와 DOM요소에 대해서 설명해보고자 한다.

## 시멘틱태그와 DOM의 의미

### DOM
Document Object Model로 html문서를 의미한다.

### sementic tag
html5문서 안에서 태그이름만 보고도 문서 구조에서 어떤 역할을 하는지 직관적으로 알 수 있게 하기위한 태그를 말한다. 



## 시멘틱 태그의 종류

시멘틱 태그의 구조적이용을 보면 사이트마다 다를 수는 있지만, <br>
'시멘틱'이라는 이름 그자체로 태그 명에서 사용할 수 있는 요소는 다음과 같다.


### <header></header>
머리말을 의미한다.<br> 본문(section)안에서도 사용될 수 있다.

### <nav></nav>
네비게이션의 역할이다.<br> 동일한 사이트안에서 다른 문서나 사이트로 이동할 요소의 묶음이 있을때 사용된다.

### <section></section>
주제별 컨텐츠 영역을 의미한다.<br> 문맥흐름 중 콘텐츠를 주제별로 묶어주는 역할을 하여 머리말인 header뿐 아니라 제목 태그 및 내용인 article이 자식요소로 들어올 수 있다.

### <article></article>
콘텐츠의 실질적은 내용을 의미하며, <br>해당 태그 안의 내용은 매우 독립적이다.

### <aside></aside>
본문이외의 내용을 표시하는데 사용한다.

### <footer></footer>
웹문서 끝자락에 들어가는 내용으로 사이트(제작자)의 정보를 표시한다.

### <address></address>
제작자의 주소 정보를 표시한다.

### <div></div>
위의 sementic 태그 외에 주로 컨텐츠를 묶어서 시각적 효과를 적용할때 사용한다.




## 예시
```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <header>
        <!--머리말지정 : 특정 부분의 머리말을 의미
        본문(<section></section>)안에서 사용가능-->
        <nav>
            <!--네비게이션 역할: 헤더 / 푸터의 사이트 링크모음에 사용
            동일한 사이트 안에 다른문서나 다른 사이트의 문서로 연결할 때 사용-->
        </nav>
    </header>
    <section>
        <!--주제별 컨텐츠 영역: 문맥흐름 중 콘텐츠를 주제별로 묶음 
        이안에는 머리말인 header뿐 아니라 제목태그 및 내용인 article사용-->
        <article>
            <!--콘텐츠의 실질적인 내용.
                해당태그를 이용한 내용이 독립적이면 사용가능
            -->
        </article>
    </section>
    <aside>
         <!--본문 이외의 내용표시-->
    </aside>
    <footer>
         <!--웹문서 끝자락에 들어가는 정보들 표시 (제작자 정보)-->
        <address>
            <!--제작자 정보, 연락처정보-->
        </address>
    </footer>
</body>
</html>
```
