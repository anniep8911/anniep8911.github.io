## 웹접근성이란
웹 접근성은 장애인과 비장애인 모두가 웹 사이트를 이용할 수 있게 하는 것을 말하며,
모든 사용자가 사이트의 정보 및 기능에 동등하게 접근할 수 있도록 하는 방식이다.

### 이미지 웹 접근성 
이미지는 기본적으로 <img>태그를 사용할때, alt속성을 추가하여 해당 이미지에대한 설명이필요하다.

### 비어있는 태그 
이미지를 사용하지 않고 빈 div내에 background-image를 지정했을경우, 웹 접근성과 웹표준 둘다 어긋난다. <br>
이때는 배경이미지가 적용된 div내에 텍스트로 해당 이미지에 대한 설명을 추가하고 CSS상에서 text-indent로 텍스트를 숨기거나 color를 투명으로 하여 텍스트를 숨기는 방법이있다.

### 제목
프레임 제목제공 : 프레임 혹은 아이프레임이 각 프레임을 설명하는 간단한 제목을 제공해야한다. <br>
콘텐츠 제목제공 : 콘텐츠 블록안에 제목처리가 필요하다. 만약 제목을 노출시키지 않는 디자인이라면 CSS에서 숨김처리 한다. <br>
특수기호 사용제한 : 문서 제목과 특정 제목안에 연속된 특수기회를 사용하지 않는다 (불필요한 음성 출력방지)<br>

### 레이블
입력 서식에 포커스가 맞춰진다면, 스크린리더는 '편집창'이라고만 읽게된다. 따라서 이에 대한 정보를 제공할 수 있도록 레이블이 필요하다. <br>
레이블이 시각적으로 노출되지 않으려면, title속성을 추가하여 작성하면 된다.

### 테이블
스크린 리더기가 capthion과 th유무 및 scope유무 등으로 데이터 테이블과 레이아웃 테이블을 구분하기때문에,<br> 데이터 테이블라라면 반드시 아래와 같은 사항이 준수되어야한다. <br>

<ul>
  <li>th가 존재하며 scope속성을 가진다 </li>
  <li>caption을 가진다</li>
  <li>thead에는 td태그가 올수없다</li>
  <li>tbody만 사용하거나, thead와 tbody만 있는경우 tfoot은 생략가능하다. </li>
</ul>
