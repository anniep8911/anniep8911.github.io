## 웹팩이란?
웹팩을 설명하자면, 간단히 아래와 같은 그림으로 설명할 수 있다. <br>
<img src="https://blogger.googleusercontent.com/img/a/AVvXsEhCB6xsshlFl6j-n5DUyLDtMsa16VXM2t-ksOHfoNBF0UyqvYGuBbw_0JYCoLAkCPFnUpZWWXIDSyJfUq0JELONzNii76Xd6HYfe3IF3sgroFppMFGI4JuvP0bUmJY7E6hIANQdyoEZYRrzBitLPajkNfKVMbJgzqhC2R0Q_3ckwS54MWuOroqTxV26=w573-h231" /><br>

## 환경설정
### config.js
이 상태에서 웹팩을 사용할 수 있도록 설정하보자, 설정하려면 파일을 하나 만들어야하는데, 
파일명은 반드시 'webpack.config.js'로 만들어야한다. 만들고 나면 기존에 만들었던 js파일의 아이콘과 조금은 다른 아이콘으로 파일이 생성될 것이다.
<img src="https://blogger.googleusercontent.com/img/a/AVvXsEi6c3jTM_yFDKS0TlAPpA78KVi0bOX54SZYRaL0XQepmBusEtfCDYsJHufe-u_QDyxeWYwdYgf0MS8xh8Oxz57rsOQM1UMsvcGHIkGLQSbMbITdH-lqrb7TwZ2-7vlhlu_ELlNOIx8dEp-31XLT9_au5K-DX4M2c892qzMhi3paX2jdgGHNjORkk21h=w585-h333" />

webpack.config.js에서는 아래와 같이 설정을 해주면 된다 <br>
```js
const path = require("path");
module.exports={
    // 처음 들어가는 파일명 > 지금은 index.js가 메인 파일인 격이기때문에 index.js를 설정해주었다.
    entry : './index.js',
    output: {
        //웹팩 버전으로 빌드 시 파일이름
        filename: 'main.js',
        // 웹팩 빌드 파일이 생성될 디렉터리 위치
        path: path.resolve(__dirname,'dist'),
    },
    // 모드는 developer와 production모드가 있는데, production모드가 불필요한 내용이 안나오기때문에 production모드를 활용한다. 
    mode:production,
}
```


## 웹팩사용

### javascript build

위와같이 만들면 빌드가 된다. 그러나 지금 index.js가 빈 파일이므로, <br>
웹팩 활용을 위해 util.js라는 파일을 만들고 아래와 같이 파일에 내용을 추가했다.
```js
function hello(){
   console.log('함수내보내기');
}
var add  = '변수내보내기!';

export {hello, add};
```

그 다음, import작업을 index.js(메인이 되는 파일)에 와서 작성한다.
```js
import {hello,add} from './util.js';
hello();
console.log(add);
```

자, 이제 이것들을 webpack화 시키면된다. 이렇게 만들고 나서는 항상 webpack build를 해줘야 하는데, webpack build의 명령어를 간소화 시키기 위해서 package.json파일에 다음과 같이 설정을 해주었다.
